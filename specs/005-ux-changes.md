# 005 — 회의 결정(v0.2) 반영 UX 변경 계획서

> 입력: "고찌봄 결정 정리 v0.2". 목적: 결정사항을 현재 FE(`gojjibom-fe`, commit `194cb34`)에 어떻게 반영할지.
> 원칙: 앱 안 해치기(회귀 최소) · UI 통일 유지 · 지금은 mock, BE는 계약(§specs/003·004) 나오면 연동.

## 0. 요약
회의에서 UX에 실제로 영향을 주는 결정은 6가지: **① 사진 기반 복약 확인, ② 약 교체+복약 이력, ③ QR 등록(코드만)+약 확인, ④ 식사시간 기반 알람(알람시간 입력 없음), ⑤ 식사시간 변경은 대표자만, F 감시→안심 톤/동의.**
이 중 **화면·플로우·카피는 FE 단독(mock)으로 대부분 구현 가능**하고, **역할/권한·서버 fetch·알람 재계산·자동 이력이동·변경 알림**은 BE가 필요하다. 데모 임계경로(P0)는 **사진 확인 강화 + 식사시간 파생 표시 + 안심 톤/동의**로, 전부 FE만으로 가능.

## 2. 화면·플로우 변경안 (라우트별)

### A. 복약 확인 = 사진 (①·SAF-02·SAF-03)
- `/senior/dose`: 1차 액션을 **"약 확인하고 사진 찍기"**로. `/senior/camera`로 이동(현행 유지).
- `/senior/camera`: **유형(봉지/약통)에 따라 안내 문구·예시**를 분기.
  - 봉지: "이번에 드실 봉지를 찍어주세요."
  - 약통: "**방금 비운 그 시간 칸**이 보이게 약통을 찍어주세요." (칸이 빈 모습 = 신뢰 강화)
- `/senior/camera` **촬영 직후 자가 확인 스텝(신설)**: 찍은 사진을 크게 보여주며 "**보이는 사진이 약이 맞나요?**" [네, 맞아요 / 다시 찍기]. **'맞아요'를 눌러야 가족에게 전송**된다. (SAF-03 핵심 완화 — 엉뚱한 사진 방지, 확인 주체는 어르신 본인)
- `/senior/done`·`/caregiver`: 사진 썸네일 + "○시 칸 비움 확인" 라벨. **하단에 "사진은 참고용이며 복용 증명은 아니에요" 문구(SAF-03).**
- 결정 필요: **사진 필수 여부**(권장: 약통 유형은 사진 기본, "사진 없이"는 약통 유형에서 눈에 덜 띄게).

### B. 복약 이력 (②·SAF-07·ALM-03)
- **신설** `/senior/history`, `/caregiver/history`(또는 `/caregiver/pills` 확장 탭): 지난 조제 카드 목록(표시이름·복용기간·조제번호·약국). 진행 중/종료 구분.
- 교체·소진 시 "지난 약은 이력으로 옮겨졌어요" 안내. (자동 이동 판정은 BE, FE는 상태 배지 렌더)

### C. QR 등록 + 약 확인 (③·PRV-01·SAF-06)
- `/senior/register` 스캔 성공 → **`/senior/confirm-meds`(신설): 서버에서 받은 약 목록 표시** → "**○○님 약이 맞나요?**" [맞아요/다시 스캔] → `/senior/connected`.
- 표시 항목(등록 데이터): 표시이름·하루 횟수·복용기준(식전/식후/취침전/공복/고정)·오프셋·1회 용량·조제일수·조제번호/일·약국. **병명·주민번호·주소·전화 미표시**(PRV-01).
- 등록에 "**약이 바뀌었어요? 다시 스캔**" 진입(재등록=교체).

### D. 식사시간 기반 알람 (④·SAF-05)
- **알람시간 입력 제거.** 모든 dose 시간을 "**저녁 식사 7:00 + 식후 30분 → 오후 7:30**" 파생 표시(현 `baselineNote` 확장, 홈/알림/확인 화면 공통).
- **신설** `/caregiver/meal-times`(대표자용): 아침·점심·저녁 식사시간 설정. 복용기준이 취침전/공복/고정시각인 약은 식사와 무관하게 별도 계산됨을 표시.

### E. 시간 변경 권한 · 대표자 (⑤·SAF-08·ALM-04)
- `/caregiver/manage`: **대표자(주보호자) 지정 + 역할 배지**(대표자/가족/복지사), 대표자 외에는 **보기 전용(잠금 아이콘)**. (하드코딩 구성원 제거는 S4에서 BE 연결)
- `/caregiver/meal-times`: 대표자만 편집 가능, 나머지 read-only. 변경 시 **"어르신·가족에게 알림이 갑니다" 확인** → 저장 → **변경 기록(로그)** 노출(`/caregiver/timeline`에 통합 or 별도).

### F. 감시→안심 톤 + 동의 (3번, 톤 미정)
- `/onboarding` 및 어르신 진입 카피를 **안심 톤**("가족이 약 챙기는 걸 도와드려요")으로. `/senior/register`(또는 신설 `/senior/consent`)에 **어르신 동의 스텝**("가족과 복약 상태를 공유하는 데 동의") 추가.
- **톤 최종 문구는 팀 확정 후 교체**(placeholder로 먼저 반영).

### G. 남는 위험 대응 (SAF-01·SAF-03)
- **SAF-03 (사진 위조/오촬영):** 핵심 완화 = **촬영 직후 "보이는 사진이 약이 맞나요?" 자가 확인 질문**(§A). 사진은 자동 신뢰가 아니라 **어르신 본인 확인을 거친 뒤** 전송·표시된다. 더해 완료/대시보드에 "사진은 참고이며 복용 증명은 아니에요" + 기존 "앱 확인은 보조 정보" 유지. 보호자 화면엔 사진 옆 **"사진이 이상해요 / 다시 요청"** 액션(선택). → 사진은 **추정**임을 UX 전반에서 일관되게 드러내 과신 방지.
- **SAF-01 (쓰러짐 vs 미복용 미구분):** 폰만으로는 둘을 구분 못함을 전제로, escalation 문구는 "**약 미확인**"까지만 표현(쓰러짐/응급 감지로 과장 금지). **가족 미등록 어르신**은 설정/도움 화면에 정부 **'응급안전안심서비스' 안내·연계**를 노출(가족 안전망이 없는 경우의 대안 경로).

## 3. 도메인/타입 변경 (`src/lib/types.ts`, mock)
- `dispensingType: 'pouch' | 'organizer'` (봉지/약통) — 카메라 안내·확인 카피 분기.
- `doseBasis: 'before_meal' | 'after_meal' | 'bedtime' | 'empty_stomach' | 'fixed'` (복용기준 확장).
- `MealTimes { breakfast, lunch, dinner }` + dose 시간은 mealTime + basis + offset 파생.
- 구성원 role에 **대표자(primary)** 플래그, `viewerOnly`.
- `PrescriptionHistory[]`(조제번호·기간·약국·상태), `changeLog[]`(무엇을·누가·언제).
- QR 등록 결과 = 위 "등록 항목"만(민감정보 제외).

## 4. 우선순위 (데모 임계경로)
- **P0 (FE 단독, mock으로 데모 가능):** A 사진 확인 강화 + 유형 분기 · D 식사시간 파생 표시 · F 안심 톤/동의 · G 카피. → 위험 SAF-02/03/05, PRV 톤 즉시 개선.
- **P1 (FE 화면 + BE 일부):** C 약 확인 스텝 · B 복약 이력 화면 · E 대표자/역할 UI + 식사시간 설정 화면(값 저장은 BE).
- **P2 (BE 중심):** QR 서버 fetch, 알람 자동 재계산, 자동 이력 이동, 변경 알림/기록, 응급 연계.

## 5. FE ↔ BE 경계
- **FE 단독(mock):** 신규 화면(이력·약확인·식사시간·동의)의 뼈대·플로우·카피, 유형 분기, 시간 파생 표시, 역할 배지/잠금 UI.
- **BE 필요:** QR→서버 약정보 fetch, 역할·권한 검증, 식사시간 변경→알람 재계산, 조제일수 소진→이력 자동 이동, 변경 알림(어르신·가족)·감사로그, 카카오 알림톡(§003). — `specs/003·004` 계약에 맞춰 연동.
